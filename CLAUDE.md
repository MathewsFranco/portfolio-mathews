## Commands

```bash
# Development
bun run dev          # all apps (Turborepo)
bun run dev:web      # web only → http://localhost:3001

# Build & type-check
bun run build        # all apps
bun run check-types  # TypeScript across all packages

# Lint & format
bun run check        # Biome: format + lint fix (write mode)
```

All commands run from the repo root. Package manager is **bun** — never use npm or yarn.

## Architecture

Turborepo monorepo with two workspaces:

- `apps/web` — the portfolio site (React 19 + Vite + TanStack Router)
- `packages/config` — shared TypeScript/Biome config
- `packages/env` — shared env validation

### `apps/web` structure

```
src/
  routes/               # TanStack Router file-based routes
    __root.tsx           # root layout (bare Outlet)
    index.tsx            # mounts PortfolioPage
  components/           # root-level shared components (loader, GradientBlinds)
  hooks/                # shared React hooks (e.g. useGithubData)
  features/portfolio/   # all portfolio UI lives here
    page.tsx             # PortfolioPage — composes all sections
    sections/            # one file per page section
    components/          # shared sub-components (e.g. SectionHeader)
    github/              # GitHub activity section components
    contact/             # contact section components
    animations.ts        # shared framer-motion variants & easings
    animated-background.tsx  # OGL WebGL shader (lazy-loaded)
  lib/
    portfolio-content.ts # all copy, data, and types (single source of truth)
    github.ts / github.types.ts  # GitHub API integration
  index.css             # design tokens (CSS custom properties) + all component styles
```

`routeTree.gen.ts` is auto-generated by the TanStack Router Vite plugin — never edit it manually.

### Data model

All portfolio content (copy, tech stack, expertise domains, experience items) lives in `src/lib/portfolio-content.ts`. Editing copy or adding technologies always starts here.

`TechItem.weight` (2–5) drives the visual grid layout and haptic intensity in `ExperienceSection`:
- `5` = 2×2 featured cell, strongest haptic
- `4` = 2×1 wide cell
- `3` = 1×1 standard cell
- `2` = 1×1 small cell, lightest haptic

### Styling

No Tailwind component classes — layout and component styles are hand-authored in `src/index.css` using CSS custom properties. Tailwind is used only for utility spacing/flex helpers.

Design tokens are defined in `:root` in `index.css` — always use `var(--c-*)` for colours and `var(--font-*)` for typography. Never hardcode colour values.

### Animations

Framer Motion throughout. Shared variants (`fade`, `easeOutExpo`) are in `animations.ts`. Sections animate on scroll using `whileInView` with `viewport={{ once: true }}`.

### Haptics

`web-haptics` (`useWebHaptics`) is used for touch feedback. `{ debug: import.meta.env.DEV }` enables audio feedback in dev so haptics can be tested on desktop.

## Always Do First

**Invoke the `/frontend-design` skill before writing any frontend code, every session, no exceptions.**

## Frontend Design Principles

Aim for distinctive, production-grade UI that avoids generic "AI slop" aesthetics. Every design decision should feel intentional and specific to this portfolio.

- **Typography**: Use characterful, unexpected font pairings — never fallback to Inter, Roboto, Arial, or system fonts without reason. Pair a distinctive display font with a refined body font.
- **Color**: Commit to a cohesive palette via CSS variables. Dominant colors with sharp accents outperform timid, evenly-distributed palettes.
- **Motion**: Prioritize high-impact moments — a well-orchestrated page load with staggered reveals creates more delight than scattered micro-interactions. Use `whileInView`, hover states, and scroll-triggered animations that surprise.
- **Composition**: Embrace asymmetry, overlap, diagonal flow, and grid-breaking elements. Use generous negative space OR controlled density — not the middle ground.
- **Backgrounds & depth**: Create atmosphere with gradient meshes, noise textures, layered transparencies, or dramatic shadows rather than flat solid colors.
- **Differentiation**: For each new component or section, ask: what makes this unforgettable? What's the one thing someone will remember?

Never converge on common choices — no purple gradients on white, no predictable card layouts, no cookie-cutter patterns that lack context-specific character.

## Code conventions

- **Formatter**: Biome — tabs for indentation, double quotes for JS strings
- **Imports**: `@/` alias maps to `src/`. Type-only imports use `import type`
- Route files use TanStack Router's `createFileRoute` / `createRootRoute`
- Lazy-load heavy dependencies (e.g. the WebGL background) with `React.lazy`
